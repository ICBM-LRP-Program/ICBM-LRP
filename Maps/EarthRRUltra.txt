/*------------------------
EditSVG points to the main file.
Second points to the linked file. For example, it makes sense to have watermap with ice and without ice equal with the exception to the iced area. When you set the "Second" watermap, you will edit both.
Mask points to the watermap you need to draw as a reference. For example, when editing land watermap it makes sense to see the normal watermap to avoid intersections and ambiguities.

Editing is quite simple:
 
left clicking on a vertex moves that vertex
left-clicking on some edge creates new vertex you can move
right-clicking on a vertex deletes that vertex
holding Shift when left-clicking creates new square segment.
Finally, pressing 'R' auto-detects segment hierarchy and saves the map.

When you work on two maps in the same time (like ice + no_ice), holding Alt when clicking will ignore the main watermap, holding Ctrl will ignore the secondary watermap.
When the watermap file is saved, you actually get two files - one the normal one to be used by game and one with the .view.svg extension added - this is to open in some editor (I use Inkscape), and generate the .png file to accompany the .svg

It differs in that is adds the black background + modifies some points to correctly cover Arctic and Antarctic.

The .png file is still required when generating the watermap from svg, as the land type information is taken from it.
-----------------------------*/

TIPS FOR CREATING OWNERSHIP MAP PNG
You must use the color code defined below. You can change the color code as long as you define it. It is very important that you DO NOT USE Anti-Aliasing brush, Use a solid pixel brush/pencil tool. Do not have any trapped/free/missing/rouge pixels. 
Pixels are 4 sided and the edges will be converted to vector. Be sure that the outside edge has a face of the pixel touching the border, not a corner only, otherwise it will crash. 
This process may take a few tries to get it right, but if you have any crashes when generating the Ownership map, you can always // out all the nations and try adding 1 at a time to find which color is causing the crash, then go check that section. Be sure to delete the generated files in Documents/icbm/maps/... if you need to use this method.

NOTE: You can change the names of the nations on this step, however, if you do so this will impact your next steps in generating the AICACHE.
